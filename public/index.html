API 文档

简介

TmdbCacheX 是一个 TMDB API 缓存代理服务。将请求转发至 TMDB API 并自动缓存响应，减少重复调用，提升刮削速度。兼容所有支持自定义 TMDB API 地址的客户端（Emby、Jellyfin、Kodi 等）。

Base URL

http://你的服务器IP:3333

将客户端的 TMDB API 地址改为以上地址即可。

支持的 API 端点

GET /3/movie/{id} — 电影详情（自动附带 credits、images、videos 等）
GET /3/tv/{id} — 剧集详情（自动附带 credits、images、seasons 等）
GET /3/person/{id} — 人物详情
GET /3/search/movie?query=xxx — 搜索电影
GET /3/search/tv?query=xxx — 搜索剧集
GET /3/discover/movie — 发现电影
GET /3/discover/tv — 发现剧集
GET /3/trending/{media_type}/{time_window} — 热门趋势
GET /3/movie/popular|top_rated|now_playing|upcoming — 电影列表
GET /3/tv/popular|top_rated|on_the_air|airing_today — 剧集列表
GET /3/genre/movie|tv/list — 分类列表
GET /3/collection/{id} — 系列合集
GET /3/find/{external_id} — 外部 ID 查找
GET /3/configuration — TMDB 配置信息

图片代理

GET /img/{size}{image_path} — 自定义路径
GET /t/p/{size}{image_path} — 兼容 Emby 插件（如 StrmAssistant）

两个路径功能相同，都会代理从 image.tmdb.org 获取图片。

示例：/t/p/w500/path/to/poster.jpg 或 /img/w500/path/to/poster.jpg

常用尺寸：w92, w154, w185, w300, w500, w780, original

• 配置页的「代理图片」开关控制 API 返回的图片链接格式：开启时返回 /img/...（经本服务代理），关闭时返回 https://image.tmdb.org/t/p/...（客户端直连 TMDB）。

• 管理后台自身的图片预览不受此开关影响，始终通过本服务代理获取。

自动数据增强

请求电影或剧集详情时，代理会自动附加以下数据，无需手动指定 append_to_response：

credits, images, videos, external_ids, recommendations,
similar, keywords, watch/providers, release_dates,
content_ratings, aggregate_credits

缓存策略

  • 缓存有效期：7 天
  • 相同 URL 请求在有效期内直接返回缓存，不会请求 TMDB
  • 请求列表页时，自动在后台预取详情页（prefetch）
  • Warmer 自动定期抓取热门/趋势内容保持缓存新鲜

客户端配置示例

Emby + StrmAssistant 插件

安装 StrmAssistant 插件后，可同时代理 API 和图片：

  1. 安装 StrmAssistant 插件
  2. 进入插件设置，找到「替代 TMDB 配置」
  3. 替代 API 地址填：http://你的服务器IP:3333
  4. 替代图片地址填：http://你的服务器IP:3333
  5. API 密钥留空（使用系统默认）或填入你自己的 Key

元数据和图片都通过缓存代理，适合无法直接访问 TMDB 的网络环境。

curl 测试
curl http://你的服务器IP:3333/3/movie/550?api_key=YOUR_KEY
请求格式

所有请求格式与 TMDB API 完全一致，只需将 api.themoviedb.org 替换为本服务地址。api_key 参数仍需传入你的 TMDB API Key。

认证密钥

可在配置页设置「认证密钥」，设置后客户端的 api_key 参数必须匹配此密钥才能使用代理。适用于 StrmAssistant 等插件：

  1. 在配置页设置一个认证密钥（如 my-secret-key）
  2. 在 StrmAssistant 的「API 密钥」栏填入此密钥
  3. 代理收到请求后验证密钥，通过后自动替换为真实的 TMDB Key 请求 TMDB

• 未开启认证时：无论客户端填入什么密钥（包括插件自带的 TMDB Key），代理都会自动替换为本项目的真实 TMDB Key，无需额外配置即可正常使用。

• 开启认证后：只有填写了认证密钥的请求才能通过。即使客户端填入了真实的 TMDB Key 也会被拒绝（返回 401），以防止绕过认证。

DNS 覆盖

适用于因 DNS 污染无法访问 TMDB 的用户。开启后，服务端通过外部 DNS（8.8.8.8 / 1.1.1.1）解析 TMDB 域名，绕过本地 DNS 污染。

  • 配置页提供「检测」按钮，可提前验证你的网络是否支持
  • 检测三项：DNS 服务器可达性、域名解析、TCP 连接
  • 三项全部通过才能正常使用
  • 解析结果缓存 5 分钟，不影响性能

• 如果检测未通过，说明你的网络无法直连 TMDB 服务器，需要使用 HTTP 代理。

• DNS 覆盖和 HTTP 代理可以同时开启，互不冲突。

注意事项

  • 缓存代理仅支持 GET 请求
  • 如需在内网使用图片代理，请确保客户端能访问本服务的 /img/* 或 /t/p/* 端点
  • HTTP 代理设置对所有 TMDB API 请求和图片请求生效
